THRD — Dev Machine Setup

Follow these exact steps. No extra commands have been added.

Install Google Chrome

  1. Download the .deb file to ~/Downloads
    https://www.google.com/chrome/
  2. Install Chrome: 
    cd ~/Downloads
sudo dpkg -i google-chrome-stable_current_amd64.deb

Install VS Code

  1. Download .deb file to ~/Downloads
    https://code.visualstudio.com/download
  2. Install VS Code: 
    sudo dpkg -i code_[complete filename].deb
In configuring Code: Add Microsoft apt repository for Visual Studio Code — Click Yes.

Install Miniconda

Install

mkdir -p ~/miniconda3
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh -O ~/miniconda3/miniconda.sh
bash ~/miniconda3/miniconda.sh -b -u -p ~/miniconda3
rm ~/miniconda3/miniconda.sh

Source conda env

~/miniconda3/bin/conda init
source ~/.bashrc

Install Git

sudo apt install git

Login to THRD Staff Google Account

  1. Open Google Chrome
  2. Username — thrdstaff@gmail.com
  3. Password — Ask Shubh if you do not remember
  4. Add Chrome, VS Code, and Folders to Favourites

Login to GitHub

  1. Continue with Google account

Configure SSH Key

  1. Open a terminal and run: 
    ssh-keygen -t ed25519 -C "thrdstaff@gmail.com"
    Press ENTER to all the questions
  2. Show the public key: 
    cat ~/.ssh/id_ed25519.pub
    Copy the complete key
  3. Go to github.com (ensure THRD Staff is logged in)
  4. Settings → SSH and GPG keys → New SSH key → Enter Title "" → Paste the SSH key → Click Add SSH Key

Setup Git Global Config

git config --global user.name ThrdStaff
git config --global user.email thrdstaff@gmail.com

Create THRD Directory

mkdir -p ~/Documents/thrd/codes/
cd ~/Documents/thrd/codes/

Clone Resources

IMPORTANT: Make sure to add "yes" when it asks "Are you sure you want to continue connecting".
git clone git@github.com:THRD-GIT/resources.git

Run the Setup File

cd ~/Documents/thrd/codes/resources/initial_setup
sudo    chmod +x setup.sh # enter password 

./setup.sh

Debugging help

If there is a MESA error on any laptop when running pygame, this is because the libGL driver in conda is old and cannot bind with MESA. We can delete conda's libGL so that it is forced to use the system's libGL.

To remove conda's libGL

rm -f $CONDA_PREFIX/lib/libstdc++.so*
If any Python packages are to be installed, make sure to run:
conda activate thrd
pip install library_name
