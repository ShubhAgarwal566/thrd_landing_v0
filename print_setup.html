1 Extract and Install the Driver

What This Does:

This section extracts the printer driver from a compressed file and prepares it for installation.

# Create working directory
mkdir -p ~/tsc_driver/tsc64 i mkdir -p ~/tsc_driver/tsc64

mkdir: Makes a new folder
-p: Creates parent folders if they don't exist (no error if folder already exists)
~/tsc_driver/tsc64: The location of the new folder in your home directory

Think of it like: "Create a new folder and all folders needed to get there"

In Simple English: Create a new folder called "tsc64" inside "tsc_driver" in your home directory. If the folders don't exist, create them.

# Extract the downloaded driver zip
unzip -o "/home/thrd/Desktop/Printer Driver/linux64_v1.2.13.zip" -d ~/tsc_driver/tsc64 i unzip -o [file] -d [destination]

unzip: Extracts files from a compressed .zip archive
-o: Overwrites existing files without asking
-d: Specifies the destination folder

Like opening a zip file on your computer and extracting everything to a specific folder

In Simple English: Unzip the driver file and put all the contents into the tsc64 folder we just created.

# Extract the actual driver package inside
tar -xvzf ~/tsc_driver/tsc64/Linux64_v1.2.13/barcodedriver-1.2.13-x86_64.tar.gz -C ~/tsc_driver/tsc64/ i tar -xvzf [file] -C [destination]

tar: Tool for working with archived files
-x: Extract files
-v: Verbose (show what's happening)
-z: Handle gzip compression (.gz files)
-f: Specify the file to work with
-C: Change to this directory before extracting

Like opening a compressed folder and showing you each file as it's extracted

In Simple English: Inside the zip file was another compressed file (tar.gz). This command extracts that file and shows you what's being extracted.

✅ Result: The driver files are now extracted to:
~/tsc_driver/tsc64/barcodedriver-1.2.13/

2 Install the Printer Filter

What This Does:

A "filter" is a program that converts your print job into a language the printer understands. This step installs that converter.

sudo cp ~/tsc_driver/tsc64/barcodedriver-1.2.13/rastertobarcodetspl /usr/lib/cups/filter/ i sudo cp [source] [destination]

sudo: Run command with administrator privileges
cp: Copy a file
Source: The filter file from the driver
Destination: System folder where CUPS looks for filters

Like copying a file, but you need admin permission because it goes into a system folder

In Simple English: Copy the printer filter file to the system folder where the printing system (CUPS) can find it. You need admin rights (sudo) because it's a system folder.

sudo chown root:root /usr/lib/cups/filter/rastertobarcodetspl i sudo chown root:root [file]

chown: Change ownership of a file
root:root: Set owner to "root" (system admin) and group to "root"

Like assigning who "owns" the file - in this case, the system itself

In Simple English: Set the owner of the filter file to "root" (the system administrator account). This is a security measure.

sudo chmod 755 /usr/lib/cups/filter/rastertobarcodetspl i sudo chmod 755 [file]

chmod: Change file permissions (who can read/write/execute)
755: Owner can do everything (7), group can read & execute (5), others can read & execute (5)

Sets who can use this file - everyone can run it, but only admin can change it

In Simple English: Set the file permissions so the system can run it, but only administrators can modify it.

# Verify the filter was installed correctly
ls -l /usr/lib/cups/filter/ | grep rastertobarcodetspl i ls -l [folder] | grep [text]

ls -l: List files with detailed information
|: Pipe - send output of one command to another
grep: Search for text in the output

Shows details about the filter file to confirm it was installed correctly

In Simple English: List all files in the filter folder and search for our filter file to make sure it's there and has the right permissions.

✅ What You Should See: A line showing the file with "rwxr-xr-x" permissions and "root root" as owner.

3 Set Up the PPD File

What This Does:

A PPD (PostScript Printer Description) file tells your computer about the printer's capabilities - paper sizes, print quality, etc.

sudo cp /home/thrd/Documents/thrd/codes/resources/initial_setup/TE244.ppd /etc/cups/ppd/ i sudo cp [source PPD] /etc/cups/ppd/

Copies the printer description file to the CUPS configuration directory where printer definitions are stored.

/etc/cups/ppd/: The standard location for PPD files in CUPS

Like adding your printer's manual to the system's library

In Simple English: Copy the printer description file to the folder where CUPS (the printing system) keeps all printer information files.

sudo chown root:lp /etc/cups/ppd/TE244.ppd i sudo chown root:lp [file]

root:lp: Owner is root, group is "lp" (line printer)
lp group: Special group for printing system users

Makes sure the file belongs to root but the printing system group can access it

In Simple English: Set the owner to "root" and the group to "lp" (the printer group), so the printing system can read it.

sudo chmod 644 /etc/cups/ppd/TE244.ppd i sudo chmod 644 [file]

644 permissions:
- Owner can read & write (6)
- Group can only read (4)
- Others can only read (4)

Lets everyone read the file, but only root can change it

In Simple English: Set permissions so everyone can read the printer description, but only the system administrator can modify it.

✅ Result: The PPD file is now in the right place with the correct permissions for CUPS to use it.

4 Configure Printer in CUPS

What This Does:

CUPS (Common Unix Printing System) is the software that manages all printing on Linux. This section starts CUPS and adds your printer to it.

sudo systemctl enable cups i sudo systemctl enable cups

systemctl: System control - manages services
enable: Makes CUPS start automatically when computer boots
cups: The printing service name

Like setting a program to "start automatically" when you turn on your computer

In Simple English: Set CUPS to start automatically every time you boot your computer.

sudo systemctl start cups i sudo systemctl start cups

Starts the CUPS printing service right now (doesn't wait for reboot).

Like clicking "Start" on a program

In Simple English: Start the CUPS printing service right now.

sudo systemctl status cups | grep Active i systemctl status cups | grep Active

status: Shows if service is running
| grep Active: Filters output to show only the "Active" status line

Quick check to see if CUPS is running properly

In Simple English: Check if CUPS is running and show only the status line.

sudo lpadmin -p TE244 -v usb://TSC/TE244?serial=000001 -P /etc/cups/ppd/TE244.ppd -E i lpadmin -p [name] -v [URI] -P [PPD] -E

lpadmin: Line Printer Admin - adds/modifies printers
-p TE244: Printer name (what you'll call it)
-v usb://...: Connection path (where the printer is)
-P [PPD]: Which PPD file to use
-E: Enable and accept jobs immediately

Registers your printer with CUPS and makes it ready to use

In Simple English: Add the printer to CUPS with the name "TE244", tell it where the printer is connected (USB), which PPD file to use, and enable it immediately.

lpstat -p -v i lpstat -p -v

lpstat: Line Printer Status - shows printer info
-p: Show printer status
-v: Show device URIs (connections)

Lists all printers and how they're connected

In Simple English: Show a list of all printers known to CUPS and how they're connected.

✅ What You Should See: Your TE244 printer listed with status "idle" and the USB connection path.

5 Fix Permissions for CUPS

What This Does:

This section ensures all CUPS folders have the correct permissions and adds your user account to the printer admin group.

sudo chown -R root:lp /etc/cups /var/spool/cups i sudo chown -R root:lp [folders]

-R: Recursive - applies to all files and subfolders
root:lp: Owner is root, group is lp (printer group)
/etc/cups: CUPS configuration folder
/var/spool/cups: Where print jobs are queued

Sets ownership for all CUPS files and the print queue folder

In Simple English: Set the owner of all CUPS configuration files and the print queue folder to root (system) and the lp (printer) group. The -R flag means this applies to everything inside these folders too.

sudo chmod -R 755 /etc/cups /var/spool/cups i sudo chmod -R 755 [folders]

-R: Recursive - applies to all files and subfolders
755: Owner can do everything, others can read and execute

Ensures everyone can read CUPS files and run programs, but only root can modify them

In Simple English: Set permissions on all CUPS folders so everyone can read and use them, but only the system administrator can change them.

sudo usermod -aG lpadmin $USER i sudo usermod -aG lpadmin $USER

usermod: Modify user account settings
-aG: Add (-a) to a Group (-G)
lpadmin: Printer administration group
$USER: Your current username

Gives you permission to manage printers without using sudo every time

In Simple English: Add your user account to the "lpadmin" group, which allows you to manage printers without needing administrator rights every time.

⚠️ Important: After running this command, you need to log out and log back in for the group change to take effect.
✅ Result: All CUPS files have correct permissions and you can now manage printers as a regular user.

6 Configure AppArmor for CUPS

What This Does:

AppArmor is a security system that restricts what programs can do. Sometimes it blocks CUPS from working properly, so we need to adjust it.

sudo aa-status | grep cups i sudo aa-status | grep cups

aa-status: AppArmor status - shows security profiles
| grep cups: Filter to show only CUPS-related entries

Checks if AppArmor is restricting CUPS

In Simple English: Check if AppArmor has any security restrictions active for CUPS. If it shows "enforce mode", AppArmor is actively restricting CUPS.

sudo aa-complain /usr/sbin/cupsd i sudo aa-complain /usr/sbin/cupsd

aa-complain: Sets AppArmor to "complain mode"
Complain mode: Logs violations but doesn't block them
/usr/sbin/cupsd: The CUPS daemon (service)

Tells AppArmor to allow CUPS to work but log any security concerns

In Simple English: Tell AppArmor to allow CUPS to work freely, but still log any security warnings. This is less restrictive than enforce mode.

Optional: Temporarily Disable AppArmor

sudo systemctl stop apparmor i sudo systemctl stop apparmor

Completely stops AppArmor temporarily until next reboot.

Warning: This reduces system security. Only do this for testing.

Turn off AppArmor completely (not recommended for daily use)

In Simple English: Completely turn off AppArmor temporarily. Only do this if you're having issues and want to test if AppArmor is the cause.

⚠️ Security Note: Only disable AppArmor temporarily for troubleshooting. Re-enable it with:
sudo systemctl start apparmor
✅ Result: AppArmor won't block CUPS from accessing necessary system resources.

7 Restart and Verify

What This Does:

This section restarts CUPS to apply all changes and verifies that your printer is properly recognized.

sudo systemctl restart cups i sudo systemctl restart cups

Stops and starts CUPS again to apply all configuration changes.

Like restarting an app after changing settings

In Simple English: Restart the CUPS printing service so all the changes we made take effect.

systemctl status cups | grep Active i systemctl status cups | grep Active

Checks if CUPS restarted successfully and is running.
Look for "active (running)" in the output.

Confirms CUPS is working properly after restart

In Simple English: Check that CUPS is running properly after the restart.

lpinfo -v | grep TSC i lpinfo -v | grep TSC

lpinfo: Line Printer Info - lists available devices
-v: Show device URIs (connection paths)
| grep TSC: Filter for TSC printers

Shows if CUPS can see your TSC printer

In Simple English: List all printer devices that CUPS can see, and filter to show only TSC printers. You should see your TE244.

lsusb | grep TSC i lsusb | grep TSC

lsusb: List USB devices connected to computer
| grep TSC: Show only TSC devices

Confirms the printer is physically connected via USB

In Simple English: Show all USB devices connected to your computer and filter to show only TSC devices. This confirms your printer is plugged in and detected.

✅ What You Should See:
- CUPS status showing "active (running)"
- lpinfo showing: usb://TSC/TE244?serial=000001
- lsusb showing your TSC printer device

8 Test Print

What This Does:

The moment of truth! Let's send a test print to verify everything is working correctly.

echo "TSC TE244 Test Label" | lp -d TE244 i echo "text" | lp -d [printer]

echo: Displays text
|: Sends the text to the next command
lp: Line Printer - sends jobs to printer
-d TE244: Destination printer name

Creates text and sends it directly to your printer

In Simple English: Create a text message "TSC TE244 Test Label" and send it to the TE244 printer to print. The | symbol takes the output from echo and sends it to the lp (print) command.

✅ If Successful: Your printer will print a small label with the test message. Congratulations! Your printer is fully set up!

🎯 Optional: Raw Printing Mode

What This Does:

Raw mode allows you to send TSPL2 commands directly to the printer without any processing. Useful for advanced users who want complete control.

sudo lpadmin -p TE244_RAW -v usb://TSC/TE244?serial=000001 -m raw -E i lpadmin -p [name] -v [URI] -m raw -E

-p TE244_RAW: New printer name for raw mode
-m raw: Use raw/direct printing (no filtering)
-E: Enable immediately

Creates a second printer profile that prints without any conversion - sends commands directly

In Simple English: Create a second printer profile called "TE244_RAW" that sends print jobs directly to the printer without any processing. This is for sending raw TSPL commands.

lp -d TE244_RAW my_label.prn i lp -d TE244_RAW [file]

Prints a file containing raw TSPL commands using the raw printer profile.

.prn files: Usually contain printer-specific command codes

Sends a file with special printer commands directly to the printer

In Simple English: Print a file containing raw TSPL commands using the raw printer profile. The file should contain special commands the printer understands.

📌 Note: Raw mode is for advanced users who need to send TSPL2 commands directly. For normal printing, use the regular TE244 printer.
